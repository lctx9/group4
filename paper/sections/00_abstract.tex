\begin{abstract}
Manual unit testing remains a heavy bottleneck in software engineering, often taking up to half of a project's total development budget. Current language models generated for code completion usually suffer from compliance bias by blindly approving broken code, but no prior work has tested if interactive terminal feedback can stop this. We built an open-source two-agent setup connecting DeepSeek-V3 in a live SWE-agent CLI terminal with Llama-3.3-70B, and ran tests across all 1,552 Python tasks from TestExplora. The live terminal feedback achieved a 35.05\% Fail-to-Pass rate at Pass@1 compared to 16.60\% for the static baseline, reaching 71.59\% at Pass@3 with $p = 0.000000$ and Cliff's $\delta = 0.3018$. These results show that giving language models direct command line access helps catch real bugs before code gets pushed to production.
\end{abstract}
