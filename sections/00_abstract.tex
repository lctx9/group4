\begin{abstract}
Industrial software maintenance allocates nearly half its development budget to manual unit testing. Static language models trained on code generation suffer from compliance bias---blindly asserting defective code as correct---yet no prior evaluation has integrated interactive terminal execution feedback into repository-level test suite generation. We benchmarked an open-source dual-agent architecture coupling DeepSeek-V3 (operating via SWE-agent CLI) and Llama-3.3-70B across 1,552 Python tasks from the TestExplora benchmark. Terminal execution feedback achieved an average Pass@1 Fail-to-Pass rate of 35.05\% compared to the 16.60\% static baseline, expanding to 71.59\% at Pass@3 (paired one-tailed Wilcoxon $p = 0.000000$, Cliff's $\delta = 0.3018$, medium effect size). Our findings confirm that coupling LLMs with real-time shell execution breaks compliance bias, providing a practical foundation for proactive defect discovery in automated continuous integration pipelines.
\end{abstract}
